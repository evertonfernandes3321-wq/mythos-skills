# Mythos Skills — Worked Example

This folder is a **self-contained demonstration** of what the Mythos skill library
produces when pointed at a codebase. It exists to show the **value** of the skills:
the depth of the findings, the fixed report format, and the prioritized remediation
plan — without you having to wire anything up first.

It contains two things:

1. **`sample-target/`** — a tiny, **deliberately insecure** Node/Express app. It is a
   *demo punching bag*: every file is intentionally vulnerable so the auditors have
   something real to find.
2. **`reports/`** — **sample reports** generated by running Mythos skills against that
   target, in the library's fixed output format.

> ⚠️ **The target is intentionally vulnerable.** Do **not** deploy `sample-target/`,
> do not reuse its patterns, and do not point it at a real database. It is a teaching
> artifact only.

---

## This is synthetic — no real data, no real code

Everything here is **fabricated for the demo**:

- The app is ~90 lines of synthetic code written purely to illustrate textbook flaws.
- It contains **no real credentials, no real user data, no proprietary or private code**
  from anyone. The "secret" in `db.js` is a fake placeholder for the example, and even
  so it is **masked as `***`** wherever it appears in the reports.
- The reports describe only what actually exists in `sample-target/` — they are
  realistic in tone and format, but **honest**: no invented files, functions, or findings.

---

## What the reports show

The Mythos reports follow a **fixed format** so they are comparable across skills and
across runs:

- **Executive summary** + a **security-maturity rating**.
- **Findings** in a fixed block: Severity · Priority (P0–P3) · Confidence ·
  `file:line` (pointing at real lines in `sample-target/`) · Evidence · Impact ·
  Fix (with a concrete code example) · How to validate.
- A **consolidated table** of all findings.
- A **phased remediation plan**.
- A **final checklist**.

Three sample reports are included here:

| File | Skill | What it covers |
|------|-------|----------------|
| `reports/00-EXECUTIVE-SUMMARY.md` | (synthesis) | Scorecard, top P0/P1, Go/No-Go verdict |
| `reports/security-audit-full.md` | `security-audit-full` | End-to-end defensive audit of the target |
| `reports/injection-xss-csrf-audit.md` | `injection-xss-csrf-audit` | Focused injection / XSS / CSRF / headers audit |

> A **full** `RUN-ALL` pass applies **all ~44 skills** and would produce **44 reports**
> plus an index and synthesis. To keep this example readable, only **3 sample reports**
> are checked in here.

## How to reproduce this yourself

You only need the skills installed and the Claude Code CLI.

1. **Install the skills.** Copy the skill folders from `mythos-skills/` into your
   global skills directory:
   ```powershell
   # Windows (PowerShell), from the mythos-skills repo root
   Copy-Item .\* "$env:USERPROFILE\.claude\skills\" -Recurse -Force
   ```
   ```bash
   # macOS / Linux, from the mythos-skills repo root
   cp -r ./* ~/.claude/skills/
   ```

2. **Open Claude Code inside the target repo.** For this example that is
   `examples/sample-target/`. For your own audit, open it inside the project you
   want to analyze.

3. **Run a single skill** by invoking it directly, e.g.:
   ```
   /security-audit-full
   ```
   or
   ```
   /injection-xss-csrf-audit
   ```

4. **Or run everything at once.** Paste the contents of the repo's **`RUN-ALL.md`**
   prompt into Claude Code (with ultracode enabled). It discovers every installed
   skill, decides applicability per skill, and writes one report per skill into
   `./mythos-reports/`, plus `00-EXECUTIVE-SUMMARY.md` and `INDEX.md`.

The reports in this folder were produced by that exact flow against `sample-target/`,
then trimmed to three samples.
